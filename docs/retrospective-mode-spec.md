# 振り返りモード 機能仕様書

## 概要

ユーザーが定期的に自己振り返りを行い、成長・改善サイクルを回せる機能。
AI との対話を通じて振り返りを構造化し、履歴として蓄積する。

---

## 画面レイアウト

```
┌──────────────────────────────────────────────┐
│  [日] [週] [月] [年]          期間: 4/13〜4/19 │
├────────────────────┬─────────────────────────┤
│                    │  📋 振り返りドキュメント  │
│   AI チャット      │                         │
│                    │  気づいたこと            │
│  「今週どんな出来  │  ・...                  │
│   事がありました   │                         │
│   か？」           │  改善点                 │
│                    │  ・...                  │
│  [ユーザー入力]    │                         │
│                    │  次のどうなっていたら最高か？│
│                    │  ・...                  │
│                    │                         │
│                    │  そのためにやること・辞めること│
│                    │  ・...                  │
│                    │                         │
│                    │  ✅ 達成タスク (3件)     │
│                    │  ・タスクA              │

```

- 左ペイン: AI チャット（質問→回答の対話）
- 右ペイン: 振り返りドキュメント（リアルタイム自動更新）

---

## 振り返りの種類

| 種別 | 期間 | 起点 |
|------|------|------|
| 日次 | 当日 0:00〜現在 | 毎日 |
| 週次 | 直近 月〜日 | 毎週 |
| 月次 | 当月 1日〜月末 | 毎月 |
| 年次 | 当年 1/1〜12/31 | 毎年 |

---

## 振り返りの観点（ドキュメント構造）

```
## 気づいたこと
  （自由記述。AI の質問への回答から自動抽出）

## 改善点
  （「気づいたこと」から派生する具体的な改善アクション）

## 次のどうなっていたら最高か？
  （理想の状態・ゴールイメージ）

## そのためにやること・辞めること
  （やること / 辞めること の2列 or 箇条書き）

## 達成タスク
  （振り返り期間内に column="done" になったタスク一覧）
```

## AI 対話フロー

### 1. セッション開始

- ユーザーは **振り返りタブ** から振り返り種別（日/週/月/年）を選択し開始ボタンを押す
- ドラフト（未完了の振り返り）が存在する場合のボタン挙動は下記「ドラフト再開フロー」を参照
- AI が冒頭メッセージを送信（例: 「今週を振り返ってみましょう。どんな出来事がありましたか？」）
- 対象期間の達成タスクを自動取得し右ペインに表示

### 2. 対話中

- AI は上記4観点を順番にカバーする質問を行う
- ユーザーの回答を受け、右ペインの対応セクションをリアルタイムで更新
- 追加の深掘り質問も適宜行う
- 対話内容・ドキュメント状態は **ドラフトとして自動保存** され、ブラウザを閉じても失われない

### 3. 完了

- ユーザーが「完了」ボタンを押すか、AI が全観点をカバーしたと判断したとき
- AI が振り返り全体に対して評価コメントを生成し、ドキュメント末尾に追記
- 振り返りを DB に保存（`completedAt` を設定）し、振り返りタブの履歴一覧に追加

---

## ドラフト再開フロー

振り返りタブの開始ボタンは、同一種別のドラフト（`completedAt IS NULL`）の有無で表示が切り替わる。

### ドラフトがない場合

- ボタン: **「振り返りを始める」** （単一ボタン）
- 押下で新規セッション開始

### ドラフトがある場合

- ボタン領域が2つに分かれる:
  - **「前回の続きから再開」** （プライマリ）… 既存ドラフトを読み込みセッション再開
  - **「ドラフトを破棄して新規作成」** （セカンダリ）… 確認モーダル後、旧ドラフトを削除し新規開始
- ドラフトの最終更新日時・期間もボタン近傍に表示し、ユーザーが内容を把握できるようにする

ドラフトは振り返り種別 × 期間ごとに最大 1 件。期間が切り替わったタイミングで旧ドラフトが残っていても、別期間として扱う（種別+期間が違えば並存可）。

## 目標との関係

- 振り返りと目標（goals）を **データ上は直接リンクしない**（外部キー・タグ付けなし）
- ユーザーが必要に応じて AI アシスタントチャットから「この振り返りをもとに目標を更新して」等を指示し、AI が振り返り内容を参照して目標を書き換える運用
- これにより振り返りは純粋な「記録」として独立し、目標更新は任意のタイミングで行える

---

## 振り返り履歴タブ

- 既存タブ（カンバン・目標など）と並列に「振り返り」タブを追加
- タブ上部: 振り返り種別タブ（日/週/月/年）と開始ボタン（ドラフト有無で表示変化）
- 一覧表示: 種別アイコン + 期間 + 一言サマリー（AI 評価コメント冒頭）
- ドラフト中の振り返りは一覧先頭に「ドラフト」バッジ付きで表示
- クリックで詳細表示（完了済みは読み取り専用、ドラフトはセッション再開）

## データモデル

### Retrospective（新規テーブル: `retrospectives`）

```
id           TEXT PRIMARY KEY
type         TEXT  -- "daily" | "weekly" | "monthly" | "yearly"
periodStart  TEXT  -- ISO 日付
periodEnd    TEXT  -- ISO 日付
document     TEXT  -- JSON (各セクションのテキスト)
aiComment    TEXT  -- 完了時の AI 評価コメント
completedAt  TEXT  -- ISO 日時 (null = 進行中)
createdAt    TEXT
```

#### document JSON 構造

```json
{
  "findings":       "気づいたこと（テキスト）",
  "improvements":   "改善点（テキスト）",
  "idealState":     "次のどうなっていたら最高か（テキスト）",
  "actions":        "やること・辞めること（テキスト）",
  "completedTasks": ["タスクID", ...]
}
```

## WebSocket メッセージ（追加分）

| メッセージ型 | 方向 | 内容 |
|---|---|---|
| `retro_start` | C→S | `{type, periodStart, periodEnd, resumeDraftId?}` |
| `retro_update` | S→C | ドキュメント差分 `{section, content}` |
| `retro_complete` | C→S | 完了指示 |
| `retro_sync` | S→C | 完成した振り返り全量 |
| `retro_list` | C→S | 履歴一覧取得（ドラフト含む） |
| `retro_list_sync` | S→C | 履歴一覧全量（`completedAt` で完了/ドラフト判定） |
| `retro_discard_draft` | C→S | `{draftId}` 指定ドラフトを削除 |

## 実装コンポーネント

```
client/src/components/
  RetroPanel.tsx          # 振り返りタブ（履歴一覧）
  RetroSession.tsx        # 振り返りセッション画面（2ペイン）
  RetroDocument.tsx       # 右ペイン（ドキュメント表示）
  RetroHistory.tsx        # 履歴カード一覧
```

サーバー側 (`server.py`):
- `handle_retro_start()` — 期間・達成タスク取得、AI プロンプト構築、`resumeDraftId` 指定時はドラフト読み込み
- `handle_retro_complete()` — AI 評価コメント生成、DB 保存（`completedAt` 設定）
- `handle_retro_discard_draft()` — ドラフト削除
- 対話ターンごとに `document` をドラフトとして upsert（自動保存）
- `init_db()` に `retrospectives` テーブル追加
